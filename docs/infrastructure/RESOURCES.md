# Resource Tracking

## Overview

ResourceManager is the global singleton that tracks the game's two currencies: Madra (primary) and Gold (secondary). Both are persisted via SaveGameData.

## Madra (primary currency)
- Generated by Cycling mini-game: `base_madra_per_cycle * mouse_tracking_accuracy`
- Managed by `ResourceManager.add_madra(amount)` — clamps to stage max
- `ResourceManager.spend_madra(amount)` — returns bool, no partial deduction
- Signal: `madra_changed(new_amount: float)`
- Default: 25.0

## Gold (secondary currency)
- Earned from combat victories: `floor(base_gold * combat_mult * adventure_mult * char_mult)`
- Managed by `ResourceManager.add_gold(amount)` — uncapped
- `ResourceManager.spend_gold(amount)` — returns bool
- Signal: `gold_changed(new_amount: float)`
- Default: 0.0

### Missing Functionality

- `[HIGH]` Unify the two Madra pools — currently ResourceManager Madra (from cycling) and VitalsManager Madra (in combat) are completely disconnected. Design: total Madra pool filled by cycling with a stage-based cap. At adventure start, combat Madra draws up to FOUNDATION-based cap from total pool. Unspent combat Madra returns to total pool after adventure. This creates a natural cycle→adventure loop and gives Madra a real economy (Scripting, Elixir Making, and combat all compete for the same pool)
- `[MEDIUM]` Gold has no sinks — no merchant, no spending mechanism. Accumulates with no purpose until merchants are implemented
