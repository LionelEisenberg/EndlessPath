Interaction model

  • Drag-and-drop equip — same affordance as the existing Equipment tab. Drag a row from the left page onto a slot on the right.
  • Stack stays in inventory — equipping does NOT remove the stack from owned consumables; it just marks a slot as "draws from this stack." The greyed-out row labeled EQUIPPED on the left shows the slot/stack link.
  • Click Unequip — empties the slot. The stack on the left un-greys.
  • One stack per slot — you can equip the same consumable into multiple slots (e.g., two slots both pointing at Barely Coalesced Scale) for double-tap healing. Cooldowns are per-slot in this future model.

Data shape (future)

  • InventoryData.consumables: Dictionary[ConsumableDefinitionData, int] — owned stacks (MVP, this slice)
  • InventoryData.equipped_consumables: Dictionary[int, ConsumableDefinitionData] — slot index → definition (future)
  • InventoryManager.equip_consumable(def, slot_index) and unequip_consumable(slot_index) — parallel equip_item / unequip_item
  • ConsumableCooldownTracker rekeys from def to slot_index when slots arrive

Not shown (out of scope for this mockup)

  • Stack-splitting (e.g., assigning 3 of 12 to a slot). Slots draw from the full stack.
  • Auto-restock from a quickbar / favorites. Manual equip only.
  • Tooltip on hover — we'd reuse the existing item description panel.
